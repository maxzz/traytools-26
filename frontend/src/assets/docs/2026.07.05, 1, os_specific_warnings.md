The errors were **IDE diagnostics for the `darwin` build**, not a Windows compile failure. `go build ./...` already succeeded on your machine.

## Cause

gopls cross-checks other GOOS targets. On **darwin**, `energye/systray` implements `SetIcon`, `SetTitle`, and `SetTooltip` in `systray_darwin.go`, which requires **CGO + Cocoa**. With CGO disabled (typical on Windows), gopls skips that file, so those symbols look undefined even though your code is fine on Windows.

## Fix

Split tray code by platform, same pattern as `elevation_windows.go` / `elevation_other.go`:

- **`tray_windows.go`** — `//go:build windows` — full systray implementation
- **`tray_other.go`** — `//go:build !windows` — no-op `startTray()`

That removes the `systray` import from non-Windows builds, so darwin analysis no longer hits those missing symbols.

Behavior on Windows is unchanged: tray icon, Show/Exit menu, left-click toggle, right-click menu. On other platforms the tray is simply skipped (appropriate for a Windows-focused Wails app).
