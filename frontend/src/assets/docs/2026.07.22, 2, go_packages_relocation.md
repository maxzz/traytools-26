How to properly rename a Go package from Cursor?

---

**Cursor**

Checking Cursor/gopls guidance for renaming Go packages.

Use **gopls Rename** on the package declaration ‚Äî not on an import alias.

### Correct way
1. Open any `.go` file in the package.
2. Put the cursor on the name in `package foo` at the top of the file.
3. Run **Rename Symbol** (`F2`, or right‚Äëclick ‚Üí Rename Symbol).
4. Enter the **new import path** (directory), e.g. `traytools-26-go/backend/winhighlight`.

gopls will:
- move the package‚Äôs files into the new directory
- update `package` names when needed
- rewrite import paths across the module

### Don‚Äôt do this
- Rename on `import "‚Ä¶/foo"` or on `foo.Bar` ‚Äî that only changes the **local alias** in that file (`import bar "‚Ä¶/foo"`), not the package itself.
- Manually rename the folder in the explorer without updating imports (unless you‚Äôre doing a careful hand move).

### Limits (gopls)
- Can‚Äôt move across module boundaries
- Can‚Äôt merge into an existing package
- Can‚Äôt rename `package main`
- Subpackages stay put unless you set `"gopls": { "renameMovesSubpackages": true }` in settings

### After rename
Run `go build ./...` (and fix any leftover build tags / path references). If gopls looks stale, **Go: Restart Language Server**.

For this repo‚Äôs recent moves (`highlight` ‚Üí `winhighlight`, etc.), F2 on `package highlight` is the right tool when you want Cursor/gopls to own the move + import updates.
