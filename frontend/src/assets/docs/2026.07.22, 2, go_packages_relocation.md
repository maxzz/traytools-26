How to properly rename a Go package from Cursor?

---

**Cursor**

Checking Cursor/gopls guidance for renaming Go packages.

Use **gopls Rename** on the package declaration — not on an import alias.

### Correct way
1. Open any `.go` file in the package.
2. Put the cursor on the name in `package foo` at the top of the file.
3. Run **Rename Symbol** (`F2`, or right‑click → Rename Symbol).
4. Enter the **new import path** (directory), e.g. `traytools-26-go/backend/winhighlight`.

gopls will:
- move the package’s files into the new directory
- update `package` names when needed
- rewrite import paths across the module

### Don’t do this
- Rename on `import "…/foo"` or on `foo.Bar` — that only changes the **local alias** in that file (`import bar "…/foo"`), not the package itself.
- Manually rename the folder in the explorer without updating imports (unless you’re doing a careful hand move).

### Limits (gopls)
- Can’t move across module boundaries
- Can’t merge into an existing package
- Can’t rename `package main`
- Subpackages stay put unless you set `"gopls": { "renameMovesSubpackages": true }` in settings

### After rename
Run `go build ./...` (and fix any leftover build tags / path references). If gopls looks stale, **Go: Restart Language Server**.

For this repo’s recent moves (`highlight` → `winhighlight`, etc.), F2 on `package highlight` is the right tool when you want Cursor/gopls to own the move + import updates.
