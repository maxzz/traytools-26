**Why it was unreliable**
1. **Release builds had no DevTools** — `pnpm build` was plain `wails build`, so WebView2 had DevTools disabled and Ctrl+Shift+F12 did nothing.
2. **`OpenInspectorOnStartup` needs `-debug`** — Wails only honors that flag in debug builds; `-devtools` alone is not enough for restore-on-startup.
3. **Frontend close helper was gone** — and `ToggleDevTools` only wrote `devTools: true` without opening anything.
4. **Quit path stopped scanning** — so `init.json` could drift from the real DevTools window state.

**Fixes (matching win-watch)**
| Area | Change |
|------|--------|
| `package.json` + build scripts | `-debug -devtools` (Windows also `-ldflags "-H windowsgui"`) |
| Frontend `DevToolsShortcut` | Ctrl+Shift+F12 / I → `toggleDevTools()`, **no** `preventDefault` |
| `ToggleDevTools` | Close only when open; open stays native |
| `saveWindowOptions` | Persist `platformIsDevToolsOpen()` at quit |

Rebuild with `pnpm build` (or `wails build -debug -devtools ...`) and try Ctrl+Shift+F12 again — both open and restore-on-startup should match win-watch.
