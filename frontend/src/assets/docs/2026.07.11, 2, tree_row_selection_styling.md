Tree row selection now follows a VS Code–style pattern: a clear inner outline when the tree has focus, and a subtler highlight when it does not.

### New theme file: `frontend/src/styles/tree-selection.css`

Defines tree selection colors for light and dark mode, matching the structure in `index.css`:

- `--tree-select` / `--tree-select-foreground` — unfocused selection (screenshot 3)
- `--tree-select-focused` / `--tree-select-focused-foreground` — focused selection (screenshot 2)
- `--tree-select-border` — inner outline when focused

### Changes in `kibo-ui-tree.tsx`

- `TreeView` is focusable (`tabIndex={0}`) and uses `group/tree` so child rows can react to focus
- Clicking a row focuses the tree container
- Selected rows use:
  - **Unfocused:** `bg-tree-select` (subtle gray)
  - **Focused:** blue-tinted background, inner `ring-inset` border, medium font weight

### Import in `index.css`

```css
@import "./styles/tree-selection.css";
```

To tune colors, edit the oklch values in `tree-selection.css` — the Tailwind utilities (`bg-tree-select`, `ring-tree-select-border`, etc.) pick them up automatically.
