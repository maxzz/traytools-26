## Backend — new `backend/windowtree` package
- **`manager.go`** — a `Manager` (like `toolsmenu`) registering a `windowtree` bus group with two commands: `getTree` and `getWindowInfo`.
- **`enum_windows.go`** (Windows) — native enumeration via `golang.org/x/sys/windows` lazy DLLs (`EnumWindows` + recursive `GetWindow(GW_CHILD/GW_HWNDNEXT)`, preserving z-order). `getWindowInfo` gathers the WinSpy++-style details: caption, class (+unicode), style/exstyle, window & client rects, control ID, instance, user data, parent/owner, class atom/bytes, and process name/path (`QueryFullProcessImageName`).
- **`styles.go`** — decodes `WS_*` / `WS_EX_*` bits into readable names for the Styles tab.
- **`platform_other.go`** — synthetic demo tree so the UI works on non-Windows.
- Wired into `backend/app.go` (`windowtree.New()` + `Register`).

Handles are serialized as hex strings (`0x00010478`) to avoid JS number precision loss on 64-bit HWNDs.

## Frontend — new "Windows Tree" tab
- **`bridge/groups/windowtree.ts`** — `windowTreeBus` (`getTree`, `getWindowInfo`) + typed models; exported from `bridge/index.ts`. No Wails binding regen needed since it rides the generic `Dispatch`.
- **`store/4-windows-tree.ts`** — Valtio store for the fetched tree + selected-window details (live/mutable data).
- **`tab-windowstree/a-windows-tree-atoms.ts`** — Jotai atoms for discrete UI state (selection, filter, show-handles, hide-invisible, active props tab).
- UI in `tab-windowstree/`: a resizable **[ tree | properties ]** split. The tree (reusing the existing kibo-ui tree) shows the `Desktop windows:` hierarchy with filter, handle toggle, hide-invisible toggle, and Refresh; the properties panel has **General / Styles / Class / Process** sub-tabs.
- Registered in `1-pages.tsx` and added a **View → Show Windows Tree...** menu item that switches to the tab and refreshes.

As requested, it's a tab (not a separate window), and uses Jotai/Valtio for state.
