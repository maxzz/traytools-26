## New page to edit the Tools menu
- A new page `PageToolsEditor` in `frontend/src/components/2-main/tab-tools-editor/`, registered as the `tools-editor` tab in `1-pages.tsx`.
- It's opened from the **Tools â†’ Edit Tools Menuâ€¦** entry I added to the menubar (`1-1-menu-tools.tsx`).
- The editor is a recursive tree editor (`1-tools-node-editor.tsx`): each node is a **Command**, **Submenu**, or **Separator**. You can edit `menuName`, `cmdLine`, `cmdArgs`, `cmdWhat` (Relative/Absolute/Registry), `cmdPlat`, and `hotKey`, add/delete nodes, reorder them (up/down), and nest submenus.

## Defaults correspond to the existing tools.json
- `DEFAULT_TOOLS_CONFIG` in `store/5-tools-editor.ts` is a faithful transcription of `tools/tools.json` (Registry, Folders, Utilities, Web links, separators, the `F4` hotkey, etc.). A **Reset** button restores these defaults.

## Create tools.json with a single button
- The **Save tools.json** button serializes the tree and writes the file via a new backend command.
- Backend (`backend/toolsmenu/`): added `getRaw` (returns the unparsed file text + path) and `save` (writes content) bus commands, plus `writeConfigPath`/`readRawConfig`/`saveRawConfig` in `config.go`. If a `tools.json` already exists it's overwritten in place; otherwise it's created at `<userConfigDir>/tm-template-go-26/tools/tools.json`. The saved path is shown in the toolbar.

## Load from the same location, with editor as an alternate source
- On open, the page calls `getRaw()` to load from the same search path the live menu uses (`findConfigPath`). It parses JSONC (comments/trailing commas supported, mirroring the Go stripper).

## localStorage persistence with file-missing fallback
- Every edit and every successful load is cached to `localStorage` (`traytools-26__tools__v1.0`).
- Load precedence: **on-disk file** â†’ else **cached copy** â†’ else **defaults**. So once loaded/edited, the config is preserved, and if the file is missing on a later launch, the stored version is used.
