---

Checking Wails build order — `main.go` embeds `icon.ico`, which may block regeneration.


In **Wails v2** (you’re on v2.12.0), `icon.ico` is **not** regenerated when you change `appicon.png`. Wails only creates it when the file is **missing**.

## Force regenerate

1. Update `build/appicon.png`.
2. **Delete** the existing file:
   ```powershell
   Remove-Item build\windows\icon.ico
   ```
3. Run a production build:
   ```powershell
   p build
   # or: wails build --platform windows/amd64
   ```

During **“Generating application assets”**, Wails reads `build/appicon.png` and writes `build/windows/icon.ico` (sizes 256, 128, 64, 48, 32, 16) via `winicon`. That matches what your `build/README.md` describes.

`wails dev` does **not** do this Windows packaging step, so dev won’t refresh `icon.ico`.

---

## Catch in this project

Your `main.go` embeds the `.ico` **before** Wails generates it:

```15:19:c:\y\w\2-web\0-dp\utils\traytools-26\main.go
//go:embed build/appicon.png
var icon []byte

//go:embed build/windows/icon.ico
var iconWindows []byte
```

Build order is:

1. **Generating bindings** → Go validates `//go:embed` → **needs `icon.ico` to exist**
2. **Generating application assets** → Wails creates `icon.ico` if missing

So if you delete `icon.ico` and run `p build`, you get exactly what your terminal showed:

`pattern build/windows/icon.ico: no matching files found`

### Workaround (pick one)

**A. Regenerate the `.ico` first, then build**

Using ImageMagick (if installed):

```powershell
magick convert build/appicon.png -define icon:auto-resize=256,128,64,48,32,16 build/windows/icon.ico
p build
```

**B. Temporarily drop the embed**

Comment out the `//go:embed build/windows/icon.ico` line, run `p build` once (Wails creates the file), restore the embed, run `p build` again.

**C. Overwrite in place**

If `icon.ico` already exists and you only changed `appicon.png`, Wails **won’t** overwrite it — you must delete it first (and then use A or B because of the embed).

---

## After rebuild

- **Tray icon**: comes from the embedded `icon.ico` in `main.go` — full rebuild required.
- **`.exe` icon in Explorer**: can look stale due to Windows icon cache; open the `.ico` directly or check **Properties → Details** on the exe to confirm it updated.

Right now `build/windows/icon.ico` is missing on disk, which is why your latest `p build` failed. Regenerate it with option A or B, then rebuild.

You don’t have ImageMagick installed today. The `convert.exe` on your PATH is **Windows’ disk utility**, not ImageMagick — don’t use it for icons.

## Option 1: Install ImageMagick globally (Windows)

**winget** (simplest):

```powershell
winget install ImageMagick.ImageMagick
```

Or the Q16 build:

```powershell
winget install ImageMagick.Q16
```

**Chocolatey** (you have choco 2.5.0):

```powershell
choco install imagemagick
```

After install, **open a new terminal** and verify:

```powershell
magick -version
```

Then generate the icon (same sizes Wails uses):

```powershell
magick convert build/appicon.png -define icon:auto-resize=256,128,64,48,32,16 build/windows/icon.ico
p build
```

Use `magick`, not `convert` — on Windows `convert` is the wrong tool.

---

## Option 2: No global install — Go + `winicon` (best fit for this repo)

Wails already uses `github.com/leaanthony/winicon` internally. You can generate the same `.ico` with a one-off Go program and no extra software:

```powershell
@'
package main

import (
	"bytes"
	"os"

	"github.com/leaanthony/winicon"
)

func main() {
	png, err := os.ReadFile("build/appicon.png")
	if err != nil {
		panic(err)
	}
	if err := os.MkdirAll("build/windows", 0o755); err != nil {
		panic(err)
	}
	out, err := os.Create("build/windows/icon.ico")
	if err != nil {
		panic(err)
	}
	defer out.Close()
	if err := winicon.GenerateIcon(bytes.NewReader(png), out, []int{256, 128, 64, 48, 32, 16}); err != nil {
		panic(err)
	}
}
'@ | Set-Content scripts/gen-icon.go

go run scripts/gen-icon.go
p build
```

Add `github.com/leaanthony/winicon v1.0.0` to `go.mod` if `go run` complains it’s missing (it’s only an indirect dep today).

## Option 3: Other tools (no ImageMagick)

| Tool | Notes |
|------|--------|
| **GIMP / IcoFX / RealWorld Icon Editor** | GUI — export multi-size `.ico` manually |
| **Online PNG→ICO converters** | Fine for one-off; awkward for a dev loop |
| **ffmpeg** | You have it, but multi-size `.ico` support is poor — not a good match |
| **npm `png-to-ico`** | Works if you prefer Node: `pnpm dlx png-to-ico build/appicon.png > build/windows/icon.ico` (may need size flags) |
