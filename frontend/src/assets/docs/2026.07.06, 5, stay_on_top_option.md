# Stay on top button
_Exported on 7/6/2026 at 20:57:53 PDT from Cursor (3.10.17)_

---

**User**

A "Stay on Top" button needs to be added to the options dialog. 
Clicking this button should toggle the application window's "topmost" attribute. 
This state must be saved in the application's settings appSettings, and upon startup, the setting should be checked to ensure the window's topmost flag is enabled or disabled accordingly.

---

**Cursor**

I'll explore how settings and window management are implemented so we can add a "Stay on Top" toggle that persists in `appSettings`.


Implementing the feature using the existing `showFooter` pattern: persist in `appSettings` and apply via Wails `WindowSetAlwaysOnTop` on load and change.


The "Stay on Top" toggle is wired up. Here's what changed:

### Settings storage (`1-ui-settings.ts`)
- Added `stayOnTop: boolean` to `AppSettings` (default: `false`)
- On load and whenever settings change, calls Wails `WindowSetAlwaysOnTop()` so the window state matches the saved preference
- The value is persisted to `localStorage` with the rest of `appSettings`

### Settings dialog
- Added a **Stay on top** switch in the options dialog, using the same pattern as "Show footer"
- Toggling it updates `appSettings.stayOnTop`, which triggers the window update and save

### Flow
1. **Toggle** → switch updates `appSettings.stayOnTop` → `WindowSetAlwaysOnTop()` is called → setting saved to `localStorage`
2. **Startup** → settings load from `localStorage` → `applyStayOnTop()` runs immediately so the window reflects the saved state

In Vite-only browser dev (no Wails runtime), the call is safely ignored, same as the window title logic.
