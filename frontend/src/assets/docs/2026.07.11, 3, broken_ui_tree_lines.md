Yes — fixed. The bug was in how `TreeLines` decided when to draw the current-level vertical guide.

**Root cause:** The old logic treated the current node's column the same as ancestor columns. For level-1 children, `parentPath[0]` is always `true` (the root is the only top-level node), so the code returned `null` for the current column and skipped drawing any vertical line — unless the node was the last sibling (which only got the terminating half-line). Middle siblings at any depth had gaps.

**Fix:** `TreeLines` now matches the working `TreeGuides` approach in `2-0-panel-tree.tsx`:

1. **Ancestor columns** (`index < level - 1`) — draw a full-height vertical only when that ancestor is not the last sibling (`!parentPath[index]`).
2. **Current column** — always draw the top half (parent → this row).
3. **Current column, non-last** — also draw the bottom half (this row → following siblings).
4. **Horizontal tick** — unchanged.

The windows tree should now show continuous vertical guides between siblings, like the tools menu editor tree.

The first fix only addressed half the problem. There were two deeper issues:

**1. `parentPath` was being overwritten** — `TreeNode` replaced `parentPath[level-1]` with the current node's `isLast`, wiping out whether the parent had siblings below. That broke the trunk lines that should run through children when a parent is not the last sibling.

**2. Wrong column indexing** — `TreeGuides` draws:
- **ancestor trunks** at columns `0 … depth-1` (full height while siblings follow)
- **branch connectors** at column `depth` (top half + bottom half)

`TreeLines` was drawing the branch at column `level - 1` and only checking ancestors up to `level - 2`, so the parent's trunk column was never drawn as a separate full-height line.

**Changes made:**
- Preserve `parentPath` as passed from `WindowTreeNode` (no overwrite)
- Draw ancestor trunks at columns `0 … level-1` from `parentPath`
- Draw the branch connector at column `level` (matching `TreeGuides`)
- Adjust `paddingLeft` to `(level + 1) * indent + 8` so content aligns with the new guide position
- Add 1px overlap on vertical segments so lines connect across rows with variable height

After a refresh, the windows tree should show continuous vertical trunks like the tools menu editor. If anything still looks off, let me know which level or node.
