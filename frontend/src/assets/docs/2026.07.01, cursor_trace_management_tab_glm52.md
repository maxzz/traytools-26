## What was implemented

**Go side** — new `backend/tracemanager/` package, wired into the existing command bus under the `tracemanager` group:
- `types.go` — `SectionDescription` / `StringDescription` / `TraceCall`, mirroring the legacy `tracecall_t` and section-descriptions structs.
- `manager.go` — owns the bus handlers (`getCategories`, `saveCategories`, `openRegedit`, `exportTrace`, `importTrace`, `startStream`, `stopStream`) and a streaming goroutine that emits `tracemanager:tracecall` Wails events.
- `tracecolor.go` — the legacy `TRACECOLOR_PREFIX` + 2-hex color index + `TRACECOLOR_SUFIX` encoding.
- `registry_windows.go` — reads/writes the per-component category bitmasks from `HKLM\SOFTWARE\DigitalPersona\Tracing` (what the DLL round-trip in the original did under the hood), plus `regedit` jump and `rundll32 dpocache.dll trace exp/imp`.
- `registry_other.go` — non-Windows stub with sample categories so the UI is exercisable everywhere.

`backend/app.go` instantiates `tracemanager.New()`, registers it on the bus, and calls `Start(ctx)` in `Startup`.

**Frontend side** — new `frontend/src/components/2-main/tab-tracemanager/` folder:
- `0-trace-manager-all.tsx` — the tab page: a horizontal `[trace panels | categories]` resizable split, with the left column split vertically into the per-process windows list and the trace-line view (the web equivalent of the legacy splitter + floating `CTraceWindow` dialogs, all contained in one tab).
- `c-trace-windows-list.tsx` — the per-process list with show/hide checkboxes and Start/Stop/Clear stream controls (port of `manager_listview_t` with `LVS_EX_CHECKBOXES`).
- `d-trace-window-view.tsx` — the trace-line list with the 16-color highlighting and filter/auto-scroll (port of `CTraceWindow`'s listview).
- `e-trace-checkboxes-panel.tsx` — the categories editor with collapsible sections and the Load/Save/Export/Import/Reg Keys/Reg User/Dp Reg buttons (port of `CTraceCheckboxesCtrl`).
- `b-trace-color.ts` / `a-trace-manager-atoms.ts` — color parsing and Jotai UI atoms.

State management split as requested:
- **Valtio** (`store/3-trace-manager.ts`) — the high-frequency, mutable trace state: per-process windows, live call buffers, category sections, streaming flag. `useSnapshot` gives efficient re-renders on call appends.
- **Jotai** (`a-trace-manager-atoms.ts`) — discrete UI state: selected process, filter, auto-scroll, split percent, expanded sections, plus a Valtio-bridged derived `selectedProcessAtom`.

A bridge group (`bridge/groups/tracemanager.ts` + `wails-events.ts`) exposes the bus commands and the event subscription.

The tab is registered in `1-pages.tsx`, and `2-main/index.tsx` was made to give the active tab content a bounded height (`flex-1 min-h-0`) so the resizable panels and scroll areas fill the window.
