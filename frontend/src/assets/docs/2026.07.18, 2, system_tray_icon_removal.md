Ghost tray icons usually mean the app exits without unregistering the tray icon. I'll check how this project creates and tears down the system tray.

The tray exit hook looks empty — I'll read the shutdown path and how `systray` is supposed to be torn down.

Yes — Windows leaves a ghost icon when the process exits without calling `Shell_NotifyIcon(NIM_DELETE)`. The app never calls `systray.Quit()` on shutdown; I'll fix that and wait for the tray to finish tearing down.

Yes. That ghost icon is a Windows quirk: if the process exits without sending `Shell_NotifyIcon(NIM_DELETE)`, the tray keeps showing the icon until you hover it.

**Cause:** The app started the tray with `systray.Run`, but never called `systray.Quit()` on exit — so Windows never got the delete notification.

**Fix:** On shutdown we now:
1. Call `systray.Quit()` (triggers `NIM_DELETE`)
2. Wait briefly for the tray loop to finish before the process exits
3. Cover Exit, quit-on-close, and elevation restarts

Rebuild/restart the app and the tray icon should disappear immediately when you quit.
