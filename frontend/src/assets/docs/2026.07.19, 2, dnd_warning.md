**Cursor**

I'll check how that drag-and-drop flag is set and why `pnpm dev` leaves it false.

The warning is a false positive from a startup race. Wails initializes `enableWailsDragAndDrop` as `false`, then Go sets it to `true` only after WebView navigation completes — React can check before that. `main.go` already has `EnableFileDrop: true`. Removing the race-prone diagnostic.

**Why it appears:** Wails’ JS runtime starts with `enableWailsDragAndDrop: false`. Go only sets it to `true` later (on WebView `navigationCompleted`), when `main.go` has `EnableFileDrop: true`. Your Copy tab `useEffect` often runs first — especially under Vite HMR — so the check warned even though drag-and-drop was already configured correctly.

**Fix:** Removed that diagnostic. It was a temporary debug aid and is inherently racey; it does not mean `EnableFileDrop` is off. File drop still depends on `main.go` (`EnableFileDrop: true`), which is already set.
