## What I built

**Go backend** — new `backend/tracemanager/` package, wired onto the existing command bus under the `tracemanager` group (registered in `app.go`, started in `Startup`, stopped in `shutdown`):

- `types.go` / `tracecolor.go` / `format.go` — the data model (`TraceCall`, `SectionDescription`, `StringDescription`), the 16-color `'`I0D`'`-style marker parser, and the self-describing `#..section..#` text format used by the DP DLLs.
- `builtin.go` — real category descriptions (the `dpofeedb` table copied verbatim from `debugcategories_fbhook.h`) used as a fallback so the panel shows meaningful content even without DP DLLs installed.
- `manager.go` / `demo.go` — the `Manager` (bus handlers, Wails event streaming, a synthetic trace generator standing in for the "feeding thread" the original left as a TODO).
- `categories_windows.go` — the authentic round-trip: locates the DP bin dir from the registry, `LoadLibrary`s the component DLLs, and calls their `gettraceinfo`/`settraceinfo`/`get_lastknownversion` exports (with the version==200 guard, mirroring `optionizeddll_t`). Falls back to reading/writing the `HKLM\SOFTWARE\DigitalPersona\Tracing\ots_<name>` bitmasks + built-in descriptions.
- `ipc_windows.go` — the reader side of the `TM_IPC_*` shared-memory protocol (`protocol_t` in `atl_trace_media.h`): a 4 KiB mapping guarded by a mutex and two events, decoding the packed `tracecall_t`. This captures real traces if a producer exists.
- `actions_windows.go` — the `regeditjump` and `rundll32 dpocache.dll trace exp/imp` button actions.
- `platform_other.go` — non-Windows stubs (in-memory categories from the built-in tables) so the UI is fully exercisable anywhere.

**Frontend** — new `frontend/src/components/2-main/tab-tracemanager/` folder:
- `0-trace-manager-all.tsx` — the tab: a resizable `[ trace panels | categories ]` split, panels column split into the per-process list (top) and the selected window's trace view (bottom). Wires up the Wails event subscriptions and initial load.
- `c-trace-windows-list.tsx` — per-process list with show/hide checkboxes and Start/Stop/Clear (port of `manager_listview_t`).
- `d-trace-window-view.tsx` — trace lines with 16-color highlighting, filter, and auto-scroll (port of `CTraceWindow`).
- `e-trace-checkboxes-panel.tsx` — collapsible category sections with Load/Save/Export/Import/Reg-User/Dp-Reg buttons (port of `CTraceCheckboxesCtrl`).
- Plus `a-trace-manager-atoms.ts` and `b-trace-color.ts`.

**State management, as requested:**
- **Valtio** (`store/3-trace-manager.ts`) — the high-frequency, mutable data: per-process windows, live call buffers (capped), and the category sections.
- **Jotai** (`a-trace-manager-atoms.ts`) — discrete UI state: selected process, filter, auto-scroll/color toggles, expanded sections, and the two split layouts (persisted via `atomWithStorage`).

A bridge group (`bridge/groups/tracemanager.ts` + `bridge/wails-events.ts`) exposes the bus commands and the typed event subscription.
