## How it works

The store now keeps two reference points:

- **`baseline`** — serialized `tools.json` text from the last load or save (uids stripped, `runElevated` normalized the same way as on disk)
- **`fileExists`** — whether `tools.json` exists on disk

On every config change, `syncDirty()` compares:

```typescript
serializeToolsConfig(current) !== baseline
```

| Situation | Dirty? |
|-----------|--------|
| Loaded from an existing file, no edits | **No** — shows "No unsaved changes" |
| Loaded from file, edited | **Yes** — "Unsaved changes" |
| Edited then reverted to match loaded state | **No** — badge clears automatically |
| No file on disk yet (defaults/cache) | **Yes** — always unsaved until first save |
| After Save | **No** — baseline updates to the saved content |

## UI

- **No unsaved changes** (muted) when a file exists and the current state matches the baseline
